# MVPタスク分解 & フェーズ分解

## フェーズ概要

```
Phase 0: 環境構築・基盤整備           ──  1週間
Phase 1: コアデータ管理 (CRUD)        ──  1.5週間
Phase 2: AI連携・カード生成           ──  1.5週間
Phase 3: 学習セッション・復習          ──  1.5週間
Phase 4: 統計・検索・設定 (Should)     ──  1週間
```

## 技術的判断ポイント

### 認証: Laravel Sanctum (SPA認証)
- **理由**: フロントとバックが分離しつつも同一ドメイン運用が可能。Cookie ベースでトークン管理が不要。個人利用ではシンプルで安全。

### AI連携: 同期処理 → Queue移行
- **Phase 2初期**: 同期的にAI APIを呼び出し、レスポンスを直接返す
- **将来**: 生成時間が長い場合はLaravel Queue + ポーリング or WebSocket へ移行

### スケジューラ: SM-2ライク簡易実装
- **Phase 3**: テスト可能な純粋関数として実装し、将来のFSRS差し替えに備える
- **設計**: `SchedulerInterface` → `Sm2Scheduler` で戦略パターン適用
